# Tourpick360 Operating Guideline

## Fixed Goal

Tourpick360 is built for three internal goals:

1. Get Google AdSense approval.
2. Grow AdSense revenue after approval.
3. Build a search-friendly Korean domestic travel information site for Google and Naver.

These goals must guide structure, content depth, internal links, and technical SEO. They must not appear as public-facing copy.

## Public Positioning

Publicly, Tourpick360 must look and read like a useful domestic travel decision guide.

Use reader-facing language:

- domestic travel guide
- regional travel guide
- seasonal travel guide
- festival travel guide
- parking, lodging area, route, budget, rainy-day backup
- checklist, comparison, decision criteria

Do not show internal language:

- AdSense approval
- approval content
- approval roadmap
- revenue purpose
- monetization target
- admin or management wording
- under construction

## Site Structure Rules

- No separate admin page.
- Every visible card should link to a real useful page.
- No empty category pages.
- Each page needs one clear H1, unique title, unique description, and useful internal links.
- Main page should act as a travel portal, not a blog list.
- Category pages should help readers choose: region, season, travel type, parking, lodging area, and route.
- Articles should include a practical checklist, FAQ, and related guides.

## Content Rules

Each article should answer a real travel decision:

- Where should I stay?
- Where can I park?
- What route is less tiring?
- What changes when it rains?
- What should families, couples, or parents check first?
- What should I confirm before booking?

Prefer useful, verifiable information over generic descriptions.

## SEO Rules

- Google and Naver should be able to crawl normal HTML links.
- Avoid JavaScript-only navigation.
- Use descriptive anchor text.
- Use descriptive image alt text.
- Keep sitemap, robots, canonical, and 404 behavior healthy.
- Add structured data where appropriate, especially WebSite and Article.

## API/Data Plan

Before approval, manually verified content is safer than thin automated API pages.
After approval, APIs can support updates for festivals, parking, tourist spots, images, and coordinates.

Recommended data sources:

- Korea Tourism Organization TourAPI
- Public Data Portal parking and local tourism APIs
- Kakao Map API for place search and coordinates
- Weather API for rainy-day and seasonal warnings

Public pages should mention only useful reader information, not internal API or approval strategy.
